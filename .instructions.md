# Bug Hunting Instructions untuk Website Rental Produk

## 📋 Konteks Proyek

Ini adalah aplikasi frontend untuk **website rental produk** yang dibangun dengan:
- **Framework**: React + TypeScript
- **Build Tool**: Vite
- **Styling**: Tailwind CSS
- **UI Components**: shadcn/ui
- **Key Features**: 
  - Katalog produk dengan detail produk
  - Sistem booking multi-step (tanggal, klien, delivery, pembayaran)
  - Profil pengguna dan manajemen rental
  - Admin dashboard
  - Autentikasi
  - Status pembayaran

## 🔍 Strategi Bug Hunting Sistematis

### Step 1: Static Analysis
```bash
# Cari masalah umum
1. Unused imports
2. Missing error handlers
3. Console.error atau console.warn yang tidak ditangani
4. Any type usage yang tidak perlu
5. Validation yang hilang
```

### Step 2: Component-Level Issues
- ❌ Missing prop validation
- ❌ State management issues
- ❌ Effect dependencies incomplete
- ❌ Memory leaks di cleanup functions
- ❌ Conditional rendering logic salah

### Step 3: Integration Testing
- ❌ API call error handling
- ❌ Data flow antar components
- ❌ State synchronization
- ❌ Form submission handling

### Step 4: UX/Edge Cases
- ❌ Empty states tidak ditampilkan
- ❌ Loading states missing
- ❌ Error boundaries needed
- ❌ Keyboard navigation
- ❌ Mobile responsiveness

## 🛠️ Common Bug Patterns di React/TypeScript

### Pattern 1: Infinite Loops
```typescript
// ❌ BAD - useEffect dependency missing
useEffect(() => {
  setState({});
}, []); // forgot dependency

// ✅ GOOD
useEffect(() => {
  setState({});
}, [dependency]);
```

### Pattern 2: Stale Closures
```typescript
// ❌ BAD - closure captures old state
const handleClick = () => console.log(state);

// ✅ GOOD
const handleClick = useCallback(() => {
  console.log(state);
}, [state]);
```

### Pattern 3: Race Conditions
```typescript
// ❌ BAD - async response tidak di-cancel
useEffect(() => {
  fetchData();
}, []);

// ✅ GOOD
useEffect(() => {
  let isMounted = true;
  fetchData().then(data => {
    if (isMounted) setState(data);
  });
  return () => { isMounted = false; };
}, []);
```

### Pattern 4: Missing Error Handling
```typescript
// ❌ BAD
const data = await fetchAPI();
setState(data);

// ✅ GOOD
try {
  const data = await fetchAPI();
  setState(data);
} catch (error) {
  setError(error);
}
```

## 🎯 Quick Start untuk Agent

### Untuk mencari bugs:

1. **Start dengan Critical Areas:**
   - Booking flow (core revenue feature)
   - Authentication (security critical)
   - Payment flow (money related)

2. **Check Error Handling:**
   - Cari try-catch blocks yang kosong
   - Cek error state handling
   - Verifikasi error messages

3. **Validate Data Flow:**
   - Props passing correctly
   - State updates properly
   - API responses handled

4. **Test Edge Cases:**
   - Empty data
   - Error responses
   - Network failures
   - Missing permissions

5. **Review Types:**
   - Check for `any` types
   - Verify prop types
   - Validate return types
